# Simon Willison: Redis array type and vibe-coding at scale

## Why it matters
Simon Willison documents having Claude Code build an interactive playground for Redis's new array data type in a WASM-compiled build running in the browser. He observes that 'when vibe-coding accelerates app development, apps become more personal, more situated, and more frequent—shipping a tool is less like launching a website and more like posting on a blog.' He also notes the Zig project's policy explicitly banning LLMs for issues, PRs, and bug tracker comments—a rare counter-signal in developer tooling. Essential for practitioners tracking how AI-assisted development is changing software release cadences.

## Lede (original)
Salvatore Sanfilippo submitted a PR adding a new data type—arrays—to Redis. I had Claude Code for web build this interactive playground for trying out the new commands in a WASM-compiled build of a subset of Redis running in the browser.
