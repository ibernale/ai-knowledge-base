# Agent READMEs: An Empirical Study of Context Files for Agentic Coding

## Why it matters
This is the first large-scale empirical study of agent context files (AGENTS.md, CLAUDE.md, etc.) across 2,303 files from 1,925 repositories. The findings are immediately actionable for practitioners: developers prioritize build commands (62.3%), implementation details (69.9%), and architecture (67.7%), but critically underspecify security (14.5%) and performance (14.5%) requirements. The paper reveals these files are 'not static documentation but complex, difficult-to-read artifacts that evolve like configuration code.' This research exposes a significant gap in how teams provide guardrails for agent-written code, highlighting the need for improved tooling and standards as agentic coding becomes mainstream.

## Abstract (original)
Agentic coding tools receive goals written in natural language as input, break them down into specific tasks, and write or execute the actual code with minimal human intervention. Central to this process are agent context files ('READMEs for agents') that provide persistent, project-level instructions. In this paper, we conduct the first large-scale empirical study of 2,303 agent context files from 1,925 repositories to characterize their structure, maintenance, and content.
