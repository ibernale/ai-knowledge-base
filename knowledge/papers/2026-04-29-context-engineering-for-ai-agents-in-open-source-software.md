# Context Engineering for AI Agents in Open-Source Software

## Why it matters
First large-scale empirical study of 2,303 agent context files (AGENTS.md, CLAUDE.md, etc.) from 1,925 repositories. Finds these files are actively maintained configuration artifacts that prioritize functional context—build commands (62.3%), implementation details (69.9%), architecture (67.7%)—while non-functional requirements like security (14.5%) and performance (14.5%) are rarely specified. Critical baseline for understanding how developers actually configure AI coding agents in practice, with implications for tool designers and agent developers.

## Abstract (original)
Agentic coding tools receive goals written in natural language as input, break them down into specific tasks, and write or execute the actual code with minimal human intervention. Central to this process are agent context files ('READMEs for agents') that provide persistent, project-level instructions. In this paper, we conduct the first large-scale empirical study of 2,303 agent context files from 1,925 repositories to characterize their structure, maintenance, and content.
