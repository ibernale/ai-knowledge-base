# Verifier-Backed Hard Problem Generation for Mathematical Reasoning

## Why it matters
This paper addresses benchmark contamination in mathematical reasoning evals by proposing verifier-backed hard problem generation. As frontier models increasingly saturate existing math benchmarks (GPT-5.5 and Claude already near-ceiling on AIME/IMO-level problems), the ability to generate fresh, verifiably correct hard problems becomes critical for distinguishing model capabilities. The approach uses formal verification to guarantee problem correctness while ensuring novelty—relevant for eval designers and reasoning researchers.

Verifier-backed hard problem generation for mathematical reasoning that addresses benchmark contamination by generating fresh, verifiably correct problems.
