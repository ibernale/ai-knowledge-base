# Agent Context Files: An Empirical Study of Agentic Coding Documentation

## Why it matters
First large-scale empirical study of 2,303 agent context files from 1,925 repositories. Reveals that developers prioritize functional context (build commands 62.3%, implementation details 69.9%, architecture 67.7%) but neglect non-functional requirements like security (14.5%) and performance (14.5%). Critical reading for teams deploying coding agents at scale — the gap between what agents need and what developers provide is now quantified.

## Abstract (original)
Agentic coding tools receive goals written in natural language as input, break them down into specific tasks, and write or execute the actual code with minimal human intervention. Central to this process are agent context files ('READMEs for agents') that provide persistent, project-level instructions. In this paper, we conduct the first large-scale empirical study of 2,303 agent context files from 1,925 repositories to characterize their structure, maintenance, and content.
