# Agent READMEs: An Empirical Study of Context Files for Agentic Coding

## Why it matters
This first large-scale empirical study of 2,303 agent context files (AGENTS.md, CLAUDE.md etc.) from 1,925 repositories reveals critical gaps in how developers guide AI coding agents. The study finds context files evolve like configuration code with frequent small additions, and developers prioritize functional context (build commands 62.3%, implementation details 69.9%) but rarely specify security (14.5%) or performance requirements. This has direct implications for agentic coding workflows and highlights risks in autonomous code generation.

## Abstract (original)
Agentic coding tools receive goals written in natural language as input, break them down into specific tasks, and write or execute the actual code with minimal human intervention. Central to this process are agent context files ('READMEs for agents') that provide persistent, project-level instructions. In this paper, we conduct the first large-scale empirical study of 2,303 agent context files from 1,925 repositories to characterize their structure, maintenance, and content.
